── P1 DESIGN RULE ──

timelineH = clamp(MIN_H, laneCount × 44 + axisH + overviewH, 45vh)

• 1 lane: no overview bar (lane IS the overview). Total ≈ 60px. Detail gets ~640px on 700px panel.

• 2+ lanes: overview bar appears (32px). Height grows proportionally. Detail shrinks but stays useful.

• Spawn mid-session: CSS transition smoothly grows timeline. Overview bar fades in.

• Manual resize: user drag overrides auto-height. Preserved across lane count changes.

• Same code path — no modes, no collapse buttons, no inline embed. Just proportional allocation.
